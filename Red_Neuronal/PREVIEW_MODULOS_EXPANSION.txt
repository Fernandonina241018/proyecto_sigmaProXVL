  📊 MÓDULO 1: INTERPRETABILITY.PY
═══════════════════════════════════════════════════════════════════════════════

Funciones principales y cómo usarlas:

1. EXPLICACIÓN INTEGRAL (PUNTO DE ENTRADA PRINCIPAL)
─────────────────────────────────────────────────────────

   def explain_predictions(pipeline, X_train, X_new=None, sample_idx=0, verbose=True):
   
   USO:
   ────
   shap_results = explain_predictions(
       pipeline=mi_modelo,
       X_train=datos_entrenamiento,
       X_new=datos_nuevos,
       sample_idx=0,  # Explicar la predicción #0
       verbose=True
   )
   
   SALIDA:
   ──────
   - SHAP summary plot (importancia global)
   - SHAP force plot (explicación local de predicción)
   - Partial Dependence Plots (relación feature vs predicción)
   - Feature interactions (pares que interactúan)


2. SHAP ANALYSIS (GLOBAL + LOCAL)
─────────────────────────────────────

   a) compute_shap_values() - Calcula valores SHAP
   
      shap_results = compute_shap_values(pipeline, X_sample, X_background)
      
      Retorna:
      ├─ 'explainer': Objeto SHAP (TreeExplainer o KernelExplainer)
      ├─ 'shap_values': Array con valores SHAP
      ├─ 'base_value': Predicción promedio (baseline)
      └─ 'feature_names': Nombres de features
   
   
   b) plot_shap_summary() - Visualiza SHAP globalmente
   
      plot_shap_summary(shap_results, plot_type="bar", max_display=15)
      
      plot_type:
      - "bar" → Feature importance (suma absoluta de SHAP)
      - "beeswarm" → Distribución de SHAP values por feature
   
   
   c) plot_shap_force() - Explicación por predicción individual
   
      plot_shap_force(shap_results, row_idx=0)
      
      Muestra cómo cada feature contribuye a alejar la predicción del baseline


3. PARTIAL DEPENDENCE (RELACIÓN FEATURE VS PREDICCIÓN)
────────────────────────────────────────────────────────

   def plot_partial_dependence(pipeline, X_train, max_features=10):
   
   USO:
   ────
   plot_partial_dependence(mi_modelo, datos_entrenamiento)
   
   UTILIDAD:
   ────────
   - Responde: "¿Qué pasa con la predicción si cambio el valor de este feature?"
   - Eje X: Rango de valores del feature
   - Eje Y: Predicción promedio del modelo
   - Muestra relaciones lineales, no-lineales, e interacciones


4. ICE CURVES (INDIVIDUAL CONDITIONAL EXPECTATION)
────────────────────────────────────────────────────

   def compute_ice_curves(pipeline, X_sample, feature_idx=0, num_points=50):
   
   USO:
   ────
   ice_results = compute_ice_curves(mi_modelo, datos, feature_idx=0)
   plot_ice_curves(ice_results)
   
   UTILIDAD:
   ────────
   - Como PDP pero para CADA instancia individualmente
   - Detecta efectos heterogéneos (comportamiento diferente por grupo)
   - Líneas azules = instancias individuales
   - Línea naranja = promedio (PDP)


5. FEATURE INTERACTIONS
────────────────────────

   def compute_feature_interactions(shap_results, top_k=10):
   
   USO:
   ────
   interactions = compute_feature_interactions(shap_results)
   plot_feature_interactions(interactions)
   
   UTILIDAD:
   ────────
   - Detecta si el efecto de Feature A depende de Feature B
   - Útil para entender comportamientos complejos

Funciones principales y cómo usarlas:

   analysis = comprehensive_anomaly_analysis(

       y_test=targets_prueba,
       y_pred=predicciones_modelo,

       verbose=True
   )
   
   RETORNA:
   ────────
   ├─ 'quality': Reporte de calidad de datos
   ├─ 'outliers_train': Outliers detectados
   ├─ 'imbalance_recommendations': Estrategias para desbalance
   ├─ 'prediction_anomalies': Predicciones sospechosas
   └─ 'drift': Data drift detectado


2. DETECCIÓN DE OUTLIERS
───────────────────────────

   a) detect_outliers() - Detecta valores anómalos
   
      outliers = detect_outliers(
          X=datos,
          method="isolation_forest",  # o "lof"
          contamination=0.1,           # Porcentaje esperado
          verbose=True
      )
      
      Retorna:
      ├─ 'outlier_mask': Boolean array
      ├─ 'outlier_indices': Índices de outliers
      ├─ 'outlier_scores': Score de anomalía
      └─ 'outlier_count': Cantidad total
   
   
   b) plot_outliers() - Visualiza outliers en 2D (PCA)
   
      plot_outliers(datos, outliers['outlier_mask'])


3. REPORTE DE CALIDAD DE DATOS
────────────────────────────────

   def data_quality_report(X, y=None):
   
   Verifica:
   ✓ Valores faltantes por columna
   ✓ Filas duplicadas
   ✓ Inconsistencias de tipo de dato
   ✓ Distribución de clases (si y proporcionado)
   
   USO:
   ────
   quality = data_quality_report(datos, targets)
   print_quality_report(quality)  # Imprime reporte bonito


4. ANÁLISIS DE DESBALANCE DE CLASES
──────────────────────────────────────

   a) recommend_imbalance_handling() - Estrategias automáticas
   
      recommendations = recommend_imbalance_handling(y)
      
      Retorna estrategias según ratio:
      - ratio >= 0.8: ✓ Balanceado (sin acción)
      - ratio >= 0.3: ⚠️  Usar SMOTE + class_weights
      - ratio < 0.3:  🚨 SMOTE + Oversampling + threshold tuning
   
   
   b) apply_smote() - Crea samples sintéticos
   
      X_balanced, y_balanced = apply_smote(
          X=datos,
          y=targets,
          sampling_strategy=0.8  # Llevar minoritarios a 80% de mayoritarios
      )


5. DETECCIÓN DE ANOMALÍAS EN PREDICCIONES
───────────────────────────────────────────

   def detect_prediction_anomalies(y_true, y_pred, y_proba=None,
                                   confidence_threshold=0.6):
   
   Detecta:
   ⚠️  Predicciones correctas con baja confianza (modelo tiene suerte)
   🚨 Predicciones incorrectas con alta confianza (problema de confiabilidad)
   
   USO:
   ────
   anomalies = detect_prediction_anomalies(
       y_true=targets_reales,
       y_pred=predicciones,
       y_proba=probabilidades,
       confidence_threshold=0.7
   )
   print_anomaly_report(anomalies)


6. DATA DRIFT DETECTION
────────────────────────

   def detect_feature_drift(X_train, X_new, threshold=0.05):
   
   Usa Kolmogorov-Smirnov test para detectar cambios en distribución.
   Si X_new tiene distribución diferente a X_train, el modelo puede estar
   prediciendo en un dominio distinto.
   
   USO:
   ────
   drift = detect_feature_drift(datos_entrenamiento, datos_nuevos)
   print_drift_report(drift)
   
   INTERPRETA:
   ───────────
   - p_value < 0.05 → Diferencia significativa (DRIFT!)
   - p_value >= 0.05 → Distribución similar
