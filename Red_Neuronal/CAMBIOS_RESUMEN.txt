💾 SISTEMA DE ALMACENAMIENTO
════════════════════════════════════════════════════════════════════════════

CARPETA: modelos_guardados/

Contenido automático:

  modelo_registro.json
  ├─ Registro central de todos los modelos
  ├─ Formato: JSON estructurado
  ├─ Campos por modelo:
  │  ├─ id (modelo_001, modelo_002, ...)
  │  ├─ filename (nombre del archivo .pkl)
  │  ├─ model_key (rf, xgb, mlp, logistic, linear)
  │  ├─ dataset_name (archivo CSV original)
  │  ├─ problem_type (binary, multiclass, regression)
  │  ├─ created_at (timestamp ISO 8601)
  │  ├─ file_size_mb (tamaño del archivo)
  │  ├─ target_col (nombre de columna objetivo)
  │  ├─ metrics (accuracy, f1, auc, rmse, etc.)
  │  ├─ train_params (parámetros usados)
  │  └─ target_classes (valores posibles del target)
  │
  modelo_001_rf_temporal_20240418_143022.pkl
  ├─ Archivo pickled (.pkl) comprimido
  ├─ Contiene:
  │  ├─ pipeline (preprocessor + modelo entrenado)
  │  ├─ meta (metadatos del dataset)
  │  ├─ bootstrap_models (100 modelos para intervalos confianza)
  │  ├─ train_params (parámetros de entrenamiento)
  │  └─ saved_at (timestamp)
  │
  modelo_002_xgb_financiero_20240418_144015.pkl
  └─ Mismo formato que modelo_001
  
  ...más modelos...

🔧 INTERFAZ TÉCNICA DE ModelManager
════════════════════════════════════════════════════════════════════════════

Métodos principales:

1. list_models() → List[Dict]
   Retorna lista de modelos ordenados por fecha

2. load_model(model_id: str) → Dict
   Carga un modelo específico por ID

3. save_training(payload, dataset_name, model_key, eval_results) → str
   Guarda un entrenamiento completo

4. get_model_info(model_id: str) → Dict
   Obtiene información de un modelo sin cargarlo completamente

5. delete_model(model_id: str) → bool
   Elimina un modelo y su registro

6. print_models_table(models: List[Dict]) → None
   Imprime tabla visual del historial

7. interactive_menu() → Dict | None
   Menú interactivo para seleccionar y cargar modelo

Métodos privados internos:
   • _load_registry()
   • _save_registry()
   • _generate_model_id()
   • _generate_filename()
   • _extract_metrics()

⚡ VENTAJAS DEL SISTEMA
════════════════════════════════════════════════════════════════════════════

1. PERSISTENCIA
   → Modelos disponibles entre sesiones (días, semanas, meses)

2. AUTOMATIZACIÓN COMPLETA
   → No hay que hacer nada manualmente
   → Guardado automático sin intervención del usuario

3. METADATOS INTELIGENTES
   → Se registra automáticamente toda la información relevante
   → Fácil auditoría y reproducción

4. ORGANIZACIÓN
   → Todos los modelos en carpeta centralizada
   → Nombres descriptivos y únicos
   → Registro JSON para consultas rápidas

5. HISTORIAL VISUAL
   → Tabla legible con información clave
   → Fácil comparación de modelos

6. REUTILIZACIÓN
   → Cargar modelos en otros módulos del proyecto
   → Sin reentrenamiento → Ahorro de tiempo/recursos

7. ESCALABILIDAD
   → Sistema funciona con 1, 10, 100 modelos
   → Ordenamiento automático
   → Performance O(n) aceptable

8. BACKWARD COMPATIBLE
   → No rompe código existente
   → Nuevos parámetros opcionales
   → Funciones originales sin cambios

🔐 CONSIDERACIONES DE DISEÑO
════════════════════════════════════════════════════════════════════════════

• IDs auto-incrementados: Evita colisiones
• Timestamps ISO 8601: Estándar universal
• JSON plano: Fácil de leer y editar
• Carpeta centralizada: Organización clara
• Backward compatible: No rompe código existente
• Sin dependencias nuevas: Solo librerías estándar (joblib, json, pathlib)
• Modular: ModelManager independiente de main.py
